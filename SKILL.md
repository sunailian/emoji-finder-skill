# Emoji Finder — 文字描述 → Emoji

根据用户提供的文字描述，找到最合适的 emoji 表情符号。

## 触发条件

- 用户说"帮我找个emoji"、"这段文字配什么表情"、"找一个表示XXX的emoji"
- 用户描述了某种情绪、场景、物品、动作，想用 emoji 表达
- 用户想确认某个 emoji 是否贴切

## 工作流程

### Step 1: 解析描述

从用户输入中提取关键语义维度：

| 维度 | 示例 |
|------|------|
| 情绪 | 开心、悲伤、愤怒、焦虑、平静、兴奋 |
| 场景 | 编程、学习、开会、运动、吃饭、旅行 |
| 物品 | 电脑、书、咖啡、手机、火箭 |
| 动作 | 思考、跑步、写字、握手、庆祝 |
| 语气 | 讽刺、真诚、俏皮、严肃、温暖 |
| 抽象概念 | 创新、突破、失败、成功、等待、成长 |

### Step 2: 匹配策略

按优先级依次尝试：

**优先级 1：精准匹配**
- 如果描述对应明确的 Unicode emoji，直接返回
- 例："火焰" → 🔥，"火箭" → 🚀，"大脑" → 🧠

**优先级 2：语义联想**
- 如果描述是抽象概念，联想相关具象 emoji
- 例："深度思考" → 🧠💭，"技术突破" → 💡🚀，"系统崩溃" → 💥🔧

**优先级 3：多候选项**
- 如果描述有歧义或多个合理选择，列出 2-3 个候选，标注适用场景

**优先级 4：组合 emoji**
- 如果单个 emoji 不足以表达，给出组合
- 例："AI Agent 开发" → 🤖⚙️💻

### Step 3: 输出格式

```markdown
## 匹配结果

**推荐**: {emoji}  — {为什么推荐，一句话理由}

**备选**:
| Emoji | 含义 | 适用场景 |
|-------|------|----------|
| {e1} | {含义} | {场景} |
| {e2} | {含义} | {场景} |

**组合建议**（如果需要）: {e1}{e2}{e3}
```

## 关键规则

1. **语义优先**：不只看字面，要理解用户想表达的意图和语气
2. **语境敏感**：同一概念在不同场景下可能需要不同 emoji
3. **给出理由**：每个推荐都要说明为什么匹配
4. **平台兼容**：优先使用广泛支持的 Unicode emoji，避免过新的 emoji
5. **不编造**：只使用真实存在的 Unicode emoji
